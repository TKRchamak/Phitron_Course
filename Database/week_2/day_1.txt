==> constraints (rules for validation):
    When table create then set the required field in the create table statement like:
        -> CREATE TABLE tableName(
            Roll INT PRIMARY KEY;
            Name VARCHAR(30) NOT NULL,
            Age INT CHECK >= 18,
            ContactNo CHAR(11) NOT NULL,
            Email VARCHAR(30) UNIQUE,
            Address VARCHAR(20) DEFAULT "no address found",
        );
    
    another approach:
        -> CREATE TABLE tableName(
            Roll INT;
            Name VARCHAR(30) NOT NULL,
            Age INT CHECK >= 18,
            ContactNo CHAR(11) NOT NULL,
            Email VARCHAR(30),
            Address VARCHAR(20) DEFAULT "no address found",

            CONSTRAINTS roll_const PRIMARY KEY(Roll),
            UNIQUE(email)
        );

    if use PRIMARY KEY, not need to use UNIQUE OR NOT NULL. it take by default that.


==> UPDATE table:
    for update sql normally gives error for safe update, to avoid that use:
        -> SET SQL_SAFE_UPDATES = 0; // safe mode off
        -> UPDATE tableName;
           SET Name = "Chamak", ContactNo = "01955406547"
           WHERE Roll = 101;
        -> SET SQL_SAFE_UPDATES = 1; // safe mode on


==> DELETE a record from table;
        -> SET SQL_SAFE_UPDATES = 0; // safe mode off
        -> DELETE from tableName
           WHERE Roll = 101;
        -> SET SQL_SAFE_UPDATES = 1; // safe mode on
